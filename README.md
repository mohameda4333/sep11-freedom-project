# SEP11 Freedom Project
by **Mohamed Aboseria**

## Context
I am a student at [HSTAT](https://www.hstat.org/) in the [Software Engineering Program](https://hstatsep.github.io/). The "Freedom Project" for SEP11 is a year-long project all about making something using JavaScript along with a third-party JS tool.

For my project, I chose to independently study **Kaboomjs** in order to help me make a **mini practice shooting game**. 

[Kaboomjs](https://kaboomjs.com/) is a third-party JS tool that makes it easier for the user to code as it provides more functions and loops that make it easier to build sprites and control them. This is a promising tool to use, especially when making a game.

## Implications
##### PROS
* Users can look and edit the code while experimenting with the game
* It helps teach users many concepts in coding like: `EventListener`, how to use the collision function, and easier game loops and functions
* Runs easy as soon as it is opened
* There are simple controls to understand and players can edit them as they play
* There is a score function to increase the score and motivate the user to continue playing the game
* 
##### CONS
* There is no way to win or lose in the game so there's no end
* The game is only available on the kaboom website itself instead of on github
* Laboomjs is a tool used mainly to make characters and control them, so it would be hard to find a job using this tool
* It is a one player game with no results but an increasing score
* The enemy sprites don't move or collide with the shooting sprite which makes the game less challenging
